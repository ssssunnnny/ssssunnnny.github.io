一、标题

在想要设置为标题的文字前面加#来表示，一个#是一级标题，两个#是二级标题，以此类推，支持六级标题。

标准语法一般在#后跟个空格再写文字。

二、字体

  • 加粗

    要加粗的文字左右分别用两个*号包起来

    **这是要加粗的文字**

    这是要加粗的文字

  • 斜体

    要斜体的文字左右分别用一个*号包起来

    这是要斜体的文字\

    这是要斜体的文字

  • 斜体加粗

    要斜体加粗的文字左右三个*号包起来

    \*这是要斜体加粗的文字***

    这是要斜体加粗的文字

  • 删除线

    要加删除线的文字左右分别用两个~~号包起来

    ~~这是要加删除线的文字~~

    这是要加删除线的文字

三、引用

在引用的文字前加>即可，引用可以嵌套，例如两个>>

>这是引用的内容

>>这是引用的内容2

这是引用的内容

这是引用的内容2

五、图片

![图片alt](图片地址 ‘’图片title’’)

图片alt就是显示在图片下面的文字，相当于对图片内容的解释，可不写

图片地址可以是相对的，也可以是绝对的

图片title是图片的标题，当鼠标移到图片上时显示的内容，可不写

  2. 插入本地图片

    因为使用hexo写博客，在网上查了一下有几种往hexo文章中插入图片的方法 ，命令都是基于上面那个命令

    • 绝对路径

      当Hexo项目中只用到少量图片时，可以将图片统一放在source/images文件夹中，通过markdown语法访问它们。

      ![](/images/图片名.jpg)

      图片既可以在首页内容中访问到，也可以在文章正文中访问到

    • 相对路径

      图片还可以放在文章自己的目录下，文章的目录可以通过配置博客根目录下的_config.yml来生成

      post_asset_folder: true

      将_config.yml文中的配置项post_asset_folder设为true后，执行命令$ hexo new post_name，在source/_posts中会生成文章post_name.md和同名文件夹post_name。将图片资源放在post_name中，文章就可以使用相对路径引用图片资源了。可以使用以下方式访问_posts/post_name/image.jpg这张照片：

      ![](post_name/image.jpg)

      ![](markdown/github.jpg)

      上述markdown的引用方式，图片只能在文章中显示，但无法在首页中正常显示。
